# DB Migrations & Seed — Skill Projet

## Doctrine Migrations
### Format
```php
final class Version20260605110000 extends AbstractMigration
{
    public function getDescription(): string { return 'Description claire'; }
    public function up(Schema $schema): void { $this->addSql('...'); }
    public function down(Schema $schema): void { $this->addSql('...'); }
}
```

### Règles
- Nom : `VersionYYYYMMDDhhmmss.php` avec `hhmmss` > à la dernière migration existante
- Toujours inclure `down()` pour rollback (sauf migrations de data fix)
- PostgreSQL : `INT GENERATED BY DEFAULT AS IDENTITY` pour les PK auto-incrément
- Index nommés : `IDX_<table>_<col>`
- FK nommées : `FK_<table>_<col>` avec `ON DELETE CASCADE|SET NULL`

### Créer une migration (si PHP dispo)
```bash
docker compose exec php php bin/console doctrine:migrations:diff
docker compose exec php php bin/console doctrine:migrations:migrate --no-interaction
```

### Créer manuellement (si PHP non dispo)
1. Lire la dernière migration pour le timestamp
2. Lire l'entité Doctrine pour le mapping exact
3. Générer le SQL PostgreSQL compatible
4. Vérifier que les contraintes FK référencent des tables existantes

## Seeds / Fixtures
Pas de `doctrine-fixtures-bundle` installé. Utiliser des **Commandes Symfony**.

### Pattern
```php
#[AsCommand(name: 'app:stock:seed', description: '...')]
class StockSeedCommand extends Command
{
    public function __construct(private EntityManagerInterface $em) { parent::__construct(); }
    protected function execute(InputInterface $in, OutputInterface $out): int
    {
        // Créer entités, persister, flush
        $out->writeln('<info>Seeded X items</info>');
        return Command::SUCCESS;
    }
}
```

### Exécuter
```bash
docker compose exec php php bin/console app:stock:seed
```

## Commandes SQL utiles PostgreSQL
```sql
-- Lister les tables
\dt
-- Schéma d'une table
\d table_name
-- Vérifier les contraintes
SELECT conname, contype FROM pg_constraint WHERE conrelid = 'table_name'::regclass;
```
